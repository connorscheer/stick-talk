# Handoff

## Active Focus

Stick Talk is a working, deployed app (stick-talk.vercel.app, auto-deploys from `main`). This was a very long session that took the app from "polished feed demo" to something much closer to a real social app: real accounts, a notifications system, follow/social graph, a Groups feature with private chat, and a persistence bug fix that was silently losing round data. The app is functionally stable, not mid-migration â€” every change in this file shipped and built cleanly.

### Accounts & identity
- Real email+password accounts via Supabase Auth (`AuthGate` in `src/App.jsx`) â€” no more no-login name gate. A signed-in account's name/home course/photo/rounds are mirrored into a shared `sticktalk:account:<user.id>` record (via `saveAccountField`) so logging in on a new device restores identity instead of starting fresh. See the "Accounts" section in `README.md` for the one-time Supabase Auth setup (Email provider is on by default; there's an optional "Confirm email" toggle to skip email verification for a small friend group).
- **Rounds persistence was broken and is now fixed.** `rounds` (what the Monthly Snapshot numbers are derived from) was never being saved anywhere â€” not localStorage, not shared storage â€” so it silently reset to empty on every page refresh. It's now loaded from the shared account record on sign-in and saved on every change (see `roundsLoaded` guard in the effects, added specifically to avoid the empty initial state stomping real data mid-load). **Anything logged before this fix is unrecoverable** â€” there was no persistence layer to have kept it.

### Social graph
- **Follows** (`sticktalk:follows`, shared): `{ [followerName]: { [followedName]: isoTimestamp } }`. `toggleFollow` in `src/App.jsx`. Followers/Following counts + list modal (`FollowListModal`) on both your own profile (`ProfileTab`) and anyone else's (`ProfileViewModal`).
- **Search** now finds people by name (every account already publishes a `profiles` entry on sign-in, so it doubles as a directory) â€” no longer filters the feed/matches by text like it originally did.
- **Notifications** (bell icon): a derived activity feed, not a stored table â€” golf claps, comments, new followers, and group join-requests all compute live from `posts`/`follows`/`groups` each render (see the `activityNotifications` useMemo). Likes/comments/follows/group-requests all carry a real timestamp now (`likedAt`, comment `time`, follow timestamp, `pendingRequests` timestamp) so the feed sorts by when things actually happened. Read-tracking is per-notification (`seenActivityIds` in localStorage) â€” opening the bell does *not* mark everything read, only tapping a specific notification does.

### Groups (new, under the "Social" nav tab)
- Bottom nav tab renamed from "Matches" to **Social**. It now has a Matches/Groups pill sub-tab switcher at the top (matches a pattern from Strava reference screenshots the user shared) â€” GPS tab was intentionally left untouched, this lives entirely inside the old Matches tab.
- Groups (`sticktalk:groups`, shared, same merge-on-poll pattern as `matches`) are **join-by-request**: `requestToJoinGroup` â†’ owner sees it in the group's own Requests panel *and* as a bell notification with inline Accept/Decline â†’ `respondToGroupRequest`.
- Once you're an accepted member, opening a group goes straight into a **private group chat** (`GroupChatModal`) â€” message list + input bar, member/request panel behind a header icon, leave/delete via a "..." menu. Non-members instead see `GroupDetailModal`, the request-to-join screen â€” the chat itself is never reachable without being an accepted member.
- Groups have their own dedicated search box *inside* the Groups sub-tab (filters "Your groups" by name locally) â€” this is deliberately separate from the global people-search in the header.
- Deliberately no group posts/photo-sharing yet, just chat + membership â€” that's what was actually asked for; a fuller Strava-style layout (Challenges tab, Events tab, timed goals) was explicitly descoped by the user.

### Header
- Real Stick Talk wordmark logo now sits where the "Clubhouse" text title used to be, top-center of the header (`StickTalkWordmark` component, image at `public/stick-talk-wordmark.png` â€” recolored from the user's original black-on-transparent export to white-on-transparent so it's visible against the dark header; if a different colorway is ever needed, redo the recolor rather than re-exporting from the user, since the current file is already the recolored version, not the original).
- Search icon moved to the top-right (next to the bell); a new **Tees currency badge** (golf-tee icon + count) sits top-left where search used to be. Tees is **display-only right now** â€” starts at 2000, persisted to localStorage, no earn/spend mechanics wired up. If the user asks to actually make Tees do something, that logic doesn't exist yet.

### Post media / scorecard (redesigned twice this session â€” read before touching)
- **Feed card sizing**: `MEDIA_BOX_HEIGHT` is now **440** (not the old locked 400) â€” bumped because the scorecard's "Differential/Rating/Slope" footer line was getting clipped by the box's `overflow: hidden`. This was verified by building a pixel-accurate static HTML replica of the scorecard markup and actually measuring it in a browser rather than guessing â€” do that again if this box ever needs to change, guessing at these values has burned multiple iterations.
- **Scorecard visual design** changed from the original "Par N" small-label look to a divided-cell look: hole number badge (muted gray, no border) above a card with a bold par digit, a hairline divider, then the score mark (still using the traditional birdie-circle/bogey-square notation via `ScoreMark` â€” unchanged). No mint borders on the cells â€” that was tried and explicitly reverted after the user provided a reference image without borders. Spacing was condensed once (gaps/padding tightened) after user feedback that it looked too spread out.
- **Fullscreen post viewer** (`PostViewerModal`) was redesigned from "full-bleed media with floating translucent pill UI on top" (the old locked design) to a **Twitter/X tweet-detail layout**: media at a bounded height (`MEDIA_LARGE_HEIGHT = "64vh"`), author/caption/actions/reply stacked *below* it in normal document flow, not overlaid. This was explicitly requested by the user with a Twitter screenshot as reference â€” the old floating-pill design is gone, don't reintroduce it.
- **Zoom is now real two-finger pinch-to-zoom + single-finger pan** on photos (manual touch handling, `touchAction: "none"`, continuous scale 1â€“4), not the old stepped tap-to-cycle (1x/1.8x/2.6x). Scorecards are **not zoomable** â€” they render at natural (uncapped) height in the fullscreen viewer instead, since forcing them into the same fixed-height zoom box as photos is exactly what caused the clipping bugs earlier this session. If scorecard clipping ever comes back, check whether someone re-added the zoom-box wrapper around `<Scorecard large />`.
- A real bug got introduced and fixed along the way: removing the feed card's tap-to-open wrapper for scorecard-only posts (while fixing fullscreen clipping) accidentally broke tapping into scorecard posts from the feed entirely. Feed cards must keep their `Item` wrapper (tap â†’ `onOpen`); only the fullscreen-viewer render path skips it.

### Skills installed in this repo (committed, travel with the repo)
- `.claude/skills/` + `.agents/skills/`: Emil Kowalski's animation/design skills (`emilkowalski/skills`) and UI/UX Pro Max (`ui-ux-pro-max-cli`, `nextlevelbuilder/ui-ux-pro-max-skill`) â€” both installed via `npx skills@latest add ...` / `npx ui-ux-pro-max-cli init`. UI/UX Pro Max's own CLI (Python-based search over its style/color/UX database) targets React Native by default; this is a web app, so its Quick Reference checklists were used directly instead of the CLI. One concrete fix that came out of that audit: `headerIconBtn` and `iconBtn` (every modal's close "X") were both under the 44Ă—44px touch-target minimum â€” fixed, `iconBtn` now uses a negative margin so the visible icon didn't shift while the tap target grew.

Known-good conventions to keep following:
- All shared feed data (posts/profiles/matches/follows/groups, plus per-account identity) goes through Supabase via `loadShared`/`saveShared` â€” never `window.storage` (that's a Claude-artifact-only API, silently broken in the real deployed app). GHIN data is still device-local `localStorage` only, not account-synced.
- Styling lives in the single `styles = {...}` object near the bottom of `src/App.jsx` â€” edit the relevant key there, don't add ad-hoc inline styles at call sites.
- Workflow: after any change, `npm install && npm run build` in the repo to verify, then commit + push to `main` directly (Vercel auto-deploys) â€” the user has asked not to be gated behind a confirmation question for routine pushes.
- Animation conventions (from the Emil Kowalski audit earlier this session): entrances use `cubic-bezier(0.23, 1, 0.32, 1)` (strong ease-out), on-screen morphing uses `cubic-bezier(0.77, 0, 0.175, 1)` (strong ease-in-out), bottom sheets use `cubic-bezier(0.32, 0.72, 0, 1)` (iOS drawer curve), all UI animations stay under 300ms, plus a global `prefers-reduced-motion` reset and a blanket `button:active { scale(0.97) }` press-feedback rule in each `<style>` block.
- This machine has no local Supabase `.env`, so nothing in this session was visually verified in a real running app â€” only via `npm run build` passing, careful code reading, and (for a couple of specific layout questions) standalone static HTML replicas measured in the browser pane. **Next session should do a real pass on the live deployed site** â€” sign up/sign in, post something, follow someone, create a group and have a second account request to join, check the fullscreen viewer's new layout and pinch-zoom on an actual phone.

No open bugs are currently tracked, but the caveat above (nothing tested against the live site) is the single biggest thing to verify next time before trusting this session's work at face value.
